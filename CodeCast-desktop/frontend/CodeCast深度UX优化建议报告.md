# CodeCast 深度 UX 优化建议报告

> 审查视角：每日使用 12 小时以上的高频用户  
> 审查范围：全部 14 个页面 + 7 个布局组件 + 9 个组件目录 + 状态管理  
> 发现问题：共 100+ 项，按优先级 P0-P3 分类

---

## P0 — 致命问题（严重阻碍日常使用，必须立即修复）

### 1. 命令面板完全无效
- **位置**：`App.tsx` 第 411 行  
- **现象**：Ctrl+K 打开命令面板后，选中任何命令都无反应（`onCommand={() => {}}`）  
- **影响**：高频用户最依赖的快速导航工具形同虚设  
- **建议**：实现 onCommand 路由，接入页面导航、会话切换、主题切换等命令

### 2. Drawer 抽屉面板从未渲染（死代码）
- **位置**：`Drawer.tsx` 包含 FileTree、GitPanel、MCPPanel、MemoryPanel 四个完整标签页  
- **现象**：整个 Drawer 组件已实现但从未被 import 到 WorkspaceFrame 或 App.tsx 中  
- **影响**：文件浏览、Git 状态、MCP 管理、记忆搜索——所有功能用户完全无法访问  
- **建议**：集成到布局中，作为可折叠的左侧/底部面板

### 3. 聊天流式输出时自动滚动失效
- **位置**：`ChatPage.tsx` 第 46-53 行  
- **现象**：流式生成时，auto-scroll 只在 `messages.length` 增加时触发（即新消息出现时），而非流式内容更新时。用户在长回复生成过程中必须手动不断向下滚动  
- **影响**：每天都在经历的折磨——看着文字在屏幕外生长，手动滚轮跟不上  
- **建议**：用 `isNearBottom` ref 追踪滚动位置，在距底部 <60px 时持续自动滚动

### 4. 消息操作全部缺失：无复制/编辑/重新生成/删除
- **位置**：`MessageItem.tsx` / `MessageStream.tsx`  
- **现象**：用户消息零交互——无法复制、编辑、删除。AI 消息的弹出菜单（Preview/Diff/Terminal）回调全部未接线（`onClick={undefined}`），点击无任何反应  
- **影响**：无法一键复制代码片段，无法修改错误提示后重新提交，无法重新生成不满意的回答  
- **建议**：  
  - 每条消息 hover 显示"复制"按钮  
  - 用户消息添加"编辑"功能（点击后填充到输入框）  
  - 最后一条 AI 消息添加"重新生成"  
  - 接线现有 Popover 回调或移除死按钮

### 5. 流式生成时缺少显眼的"停止"按钮
- **位置**：`ChatPage.tsx` + `Composer.tsx`  
- **现象**：取消按钮是 Composer 底部一个极小的 "x"，且始终可见（不只在流式时显示）  
- **影响**：AI 生成跑偏时，用户必须费力寻找隐藏的取消按钮  
- **建议**：流式时在 Composer 上方显示居中的"停止生成"按钮，非流式时隐藏

### 6. 所有页面表单状态在导航时丢失
- **位置**：CastEmailPage、CastWritingPage、CastTranslationPage、CastKnotePage、CastSchedulePage、CastToolsPage  
- **现象**：所有输入框使用本地 `useState`，切换页面后内容全部消失  
- **影响**：半写好的邮件、部分填好的表单、精心组织的翻译——每次导航全部丢失  
- **建议**：使用 sessionStorage 或 zustand draft slice 持久化草稿，页面挂载时恢复

### 7. InferenceConfig 未保存更改无警告
- **位置**：`InferenceConfigPage.tsx` 第 293 行  
- **现象**：配置了 10 分钟的 8 个分区设置后，误点"放弃更改"或返回箭头，所有工作无声丢失  
- **影响**：灾难性的数据丢失，用户信任崩塌  
- **建议**：对比 config 快照跟踪 dirty 状态，使用 `useBeforeUnload` + 确认弹窗

### 8. 快捷键在输入框中仍然触发（焦点劫持）
- **位置**：`hotkeys.ts` 第 11-19 行  
- **现象**：在聊天输入框打字时按 Ctrl+K，命令面板弹出并夺走焦点，正在编辑的消息草稿丢失  
- **影响**：每天多次丢失正在编辑的输入内容  
- **建议**：在 keydown 处理器中检查 `e.target` 是否为 INPUT/TEXTAREA

## P1 — 高优先级（显著降低使用体验）

### 9. 聊天内搜索功能缺失
- **位置**：`ChatPage.tsx`  
- **现象**：无法在长对话中搜索之前的代码片段、错误信息或决策  
- **影响**：200 条消息的会话中找一个 SQL 查询——只能手动滚动  
- **建议**：添加 Ctrl+F 搜索栏，高亮匹配的消息

### 10. 会话管理功能缺失
- **位置**：`ChatPage.tsx` / `Sidebar.tsx`  
- **现象**：无法重命名、删除、导出、置顶会话。侧边栏无右键菜单  
- **影响**：12 小时使用后，会话列表不断增长且无法整理  
- **建议**：添加侧边栏右键菜单（重命名、删除、复制、导出）

### 11. 会话切换无草稿保护
- **位置**：`Sidebar.tsx` 第 101-105 行  
- **现象**：点击其他会话立即切换，正在输入的长消息无声丢失  
- **影响**：精心构思的提示词因误点而消失  
- **建议**：切换前检测未发送消息，弹出确认或自动保存草稿

### 12. 所有 Cast 页面生成结果不可编辑
- **位置**：CastEmailPage、CastWritingPage、CastTranslationPage  
- **现象**：结果展示在 readOnly 容器中，无法直接修改  
- **影响**：AI 生成了 90% 正确的邮件但需要改一句话——只能复制→粘贴→外部编辑  
- **建议**：结果区域改为可编辑（contentEditable 或 textarea），允许用户微调

### 13. Cast 页面间无法串联工作流
- **位置**：所有 Cast 页面  
- **现象**：邮件草稿无法一键发送给"写作助手"润色，翻译结果无法直接导入邮件正文  
- **影响**：多步骤工作流需要在页面间手动复制粘贴  
- **建议**：添加"发送到..."下拉菜单，将结果管道传输到其他页面

### 14. Cast 页面大部分缺少 Ctrl+Enter 快捷键
- **位置**：CastEmailPage、CastTranslationPage、CastKnotePage、CastToolsPage  
- **现象**：仅 WritingPage 和 SchedulePage 支持 Ctrl+Enter，其余 4 个页面必须鼠标点击  
- **影响**：键盘用户每天数百次的鼠标伸手  
- **建议**：为所有页面的主要操作添加 Ctrl+Enter

### 15. 全局键盘导航完全不可用
- **位置**：全项目  
- **现象**：所有按钮使用 `onMouseEnter`/`onMouseLeave` 实现 hover 效果，Tab 键无法看到焦点状态，无 `:focus-visible` 样式  
- **影响**：键盘用户无法通过 Tab 遍历界面  
- **建议**：替换为 CSS `:hover` + `:focus-visible` 伪类，创建共享的 Interactive 基础组件

### 16. 无全局键盘快捷键切换页面
- **位置**：所有页面  
- **现象**：无法通过快捷键在邮件、写作、翻译、知识库等页面间跳转  
- **影响**：页面切换必须经过侧边栏鼠标操作  
- **建议**：注册 Ctrl+1~6 全局快捷键，或用命令面板

### 17. InferenceConfig 多个按钮无效
- **位置**：  
  - 导出按钮（第 230-233 行）— 无 onClick  
  - 测试连接按钮（第 357 行）— 无 onClick  
  - 测试模型发现按钮（第 399 行）— 无 onClick  
  - CC Switch 按钮（第 224-228 行）— 无 onClick  
  - 自定义请求头"添加标题"按钮（第 389-392 行）— 无 onClick  
  - Learn More 展开按钮（第 906-917 行）— 展开后无内容  
- **影响**：用户无法验证配置、无法导出/导入、无法添加自定义请求头  
- **建议**：逐一接线，实现 Export JSON、Test Connection、Custom Headers 编辑器

### 18. InferenceConfig 配置加载失败无错误 UI
- **位置**：`InferenceConfigPage.tsx` 第 136-143 行  
- **现象**：加载失败时 `config === null`，页面渲染空白，无任何提示  
- **影响**：用户面对空白页，以为应用卡死  
- **建议**：添加错误状态 + "重试"按钮 + 可选的"恢复默认"

### 19. 侧边栏无折叠/图标模式
- **位置**：`WorkspaceFrame.tsx`  
- **现象**：`uiSlice` 有 `sidebarOpen` 和 `toggleSidebar()`，但 WorkspaceFrame 从未使用。侧边栏始终占据固定宽度  
- **影响**：笔记本用户无法临时最大化聊天区域  
- **建议**：实现 Ctrl+B 折叠到 48px 图标栏

### 20. Toast 通知无手动关闭、无上限
- **位置**：`Toast.tsx`  
- **现象**：事件爆发时（agent 启停、编排事件、下载进度）同时显示 5-6 个 toast，每个 3 秒后自动消失，无法手动关闭  
- **影响**：重要错误通知 3 秒后消失，toast 遮挡右面板内容  
- **建议**：最多显示 3 个 + 队列；danger 类型延长到 6-8 秒；添加关闭按钮

### 21. 侧边栏"近期会话"硬限 8 条
- **位置**：`Sidebar.tsx` 第 127 行（`.slice(0, 8)`）  
- **现象**：超过 8 个会话后，旧会话在侧边栏不可见，无"查看全部"  
- **影响**：重度用户的数十个会话无法通过侧边栏找到  
- **建议**：默认显示 8 条 + "查看全部 N 个会话"展开按钮

### 22. RightPanel 叠加覆盖聊天内容
- **位置**：`RightPanel.tsx` 第 118-121 行  
- **现象**：使用 `position: absolute` 覆盖在聊天区域上方，而非推开聊天内容  
- **影响**：打开代理进度面板后无法同时阅读聊天  
- **建议**：RightPanel 打开时调整网格列宽分配专属空间

### 23. 上下文使用量进度条无警告色
- **位置**：`RightPanel.tsx` 第 260 行  
- **现象**：token 使用率 95% 时进度条仍是蓝色，无变色警告  
- **影响**：用户在不知情的情况下撞到上下文限制，工作丢失  
- **建议**：蓝色 <60%，橙色 60-85%，红色 >85%

### 24. 错误重复弹两次 Toast
- **位置**：`reportError.ts` + `App.tsx` + `useError.ts`  
- **现象**：`reportError` 推送通知触发一次 toast，`useError` 检测到 store 错误又触发一次  
- **影响**：每个错误弹两个几乎相同的 toast  
- **建议**：统一为单一路径

### 25. 无错误边界恢复 UI
- **位置**：`App.tsx` 第 414 行  
- **现象**：Sentry 错误边界 fallback 是 `<div>Something went wrong.</div>`，无重启按钮  
- **影响**：应用崩溃后完全无法自救  
- **建议**：提供"重新加载"按钮 + "报告问题"链接

## P2 — 中优先级（明显痛点，提升体验）

### 26. 聊天消息无时间戳
- **位置**：`ChatPage.tsx` 第 35 行（`createdAt: undefined`）  
- **现象**：所有消息无发送时间显示  
- **影响**：跨越数小时的长会话中，无法判断消息的时间顺序  
- **建议**：从后端传入时间戳，hover 显示"发送于 HH:MM"

### 27. 消息 key 使用数组索引
- **位置**：`ChatPage.tsx` 第 27-28 行  
- **现象**：`msg-${i}` 作为 React key，消息删除/插入时所有后续消息重新挂载  
- **影响**：长对话中的视觉闪烁和组件状态丢失  
- **建议**：确保后端消息有稳定唯一 ID

### 28. 无草稿自动保存
- **位置**：所有内容创建页面  
- **现象**：写作助手的长文档如果应用崩溃或意外关闭，30 分钟的工作全部丢失  
- **建议**：每 30 秒自动保存到 localStorage，显示"草稿保存于 HH:MM"

### 29. 每个新结果覆盖旧结果，无历史
- **位置**：CastEmailPage、CastWritingPage、CastTranslationPage  
- **现象**：v1 很好但 v2 更差——v1 永远消失了  
- **建议**：保留最近 5 个结果的历史栈，支持前后切换浏览

### 30. 邮件模板点击无确认
- **位置**：`CastEmailPage.tsx` 第 199-204 行  
- **现象**：误点模板立即替换 subject 和 body，无法撤销  
- **建议**：已有内容时弹出确认，或支持 Ctrl+Z 撤销

### 31. 日程任务删除无确认
- **位置**：`CastSchedulePage.tsx`  
- **现象**：删除按钮直接调用后端，无二次确认  
- **建议**：添加 ConfirmDialog 或 "刚刚删除"撤销 toast（5 秒窗口）

### 32. CostPage 无时间过滤和可视化
- **位置**：`CostPage.tsx`  
- **现象**：只有总成本和当前会话成本，无按日/周/月过滤，无图表  
- **影响**：无法回答"这周花了多少钱？"  
- **建议**：添加时间范围按钮 + 成本趋势折线图 + 模型成本饼图

### 33. CostPage 按模型只显示 token 数，不显示成本
- **位置**：`CostPage.tsx` 第 132-148 行  
- **现象**：看到"GPT-4: 500K tokens, Claude: 200K tokens"但不知道哪个更贵  
- **建议**：根据已知定价计算每模型估算成本

### 34. 插件卡片只显示名称和版本
- **位置**：`PluginsPage.tsx` 第 166-190 行  
- **现象**：description、author、loaded 状态、path 全部不显示  
- **建议**：展示完整信息 + 启用/禁用切换 + 配置按钮

### 35. Settings 大量后端字段未暴露
- **位置**：`SettingsPage.tsx`  
- **现象**：types.ts 中定义了 50+ 个设置字段（通知偏好、浏览器控制、Git 设置、SMTP 配置等），UI 中仅展示了约 30%  
- **建议**：按类别补充暴露，至少通知偏好、浏览器控制、Git 设置是高需求的

### 36. InferenceConfig 搜索只搜侧边栏标签
- **位置**：`InferenceConfigPage.tsx` 第 248 行  
- **现象**：搜索"proxy"或"TLS"零结果，因为 8 个侧边栏标签中没有这些词  
- **建议**：索引所有分区的字段标签和描述，匹配时自动导航 + 高亮

### 37. InferenceConfig "保存"和"应用"功能相同
- **位置**：`InferenceConfigPage.tsx` 第 294-299 行  
- **现象**：两个按钮都调用 `handleSave()`，用户困惑区别  
- **建议**：区分行为或只保留一个

### 38. InferenceConfig 禁用工具只支持单选
- **位置**：`InferenceConfigPage.tsx` 第 501-510 行  
- **现象**：`disabled_tools` 是 `string[]` 但 UI 用单个 `<select>` 只取第一个  
- **建议**：改用多选组件或 checkbox 列表

### 39. InferenceConfig 工作区文件夹图标不可点击
- **位置**：`InferenceConfigPage.tsx` 第 494-498 行  
- **现象**：文件夹图标有 `pointerEvents: 'none'`，用户必须手动输入路径  
- **建议**：绑定 `onClick` 调用 `Files.selectFolder()`

### 40. 翻译页只支持中英互译
- **位置**：`CastTranslationPage.tsx`  
- **现象**：无日语、韩语、法语等其他语言对  
- **建议**：添加更多语言对，至少支持自动检测源语言

### 41. 知识库无"新建条目"功能
- **位置**：`CastKnotePage.tsx`  
- **现象**：只支持查看、搜索、检索，无法手动添加笔记  
- **建议**：添加"新建条目"按钮 + 简单表单

### 42. 邮件页无邮箱地址验证
- **位置**：`CastEmailPage.tsx` 第 207-228 行  
- **现象**：`to` 字段接受任意字符串，包括空字符串和格式错误的地址  
- **建议**：添加邮箱正则验证 + 内联错误提示

### 43. 所有 Cast 页面输入无历史记录
- **位置**：所有 Cast 页面  
- **现象**：不记忆之前的输入，每周写工作报告的用户每次重新输入  
- **建议**：持久化每页最近 N 次输入，显示"最近使用"下拉

### 44. 邮件模板硬编码不可自定义
- **位置**：`CastEmailPage.tsx` 第 20-41 行  
- **现象**：只有 4 个固定模板，用户不能添加/编辑/删除  
- **建议**：支持自定义模板 CRUD + 持久化

### 45. TopBar 中心区域完全空白
- **位置**：`TopBar.tsx` 第 100 行  
- **现象**：不显示当前会话名称或项目名  
- **影响**：20+ 个会话中不知道当前在哪个  
- **建议**：显示当前会话名（截断），可点击打开会话选择器

### 46. 无面包屑导航
- **位置**：所有子页面  
- **现象**：只有返回箭头，无中间层级跳转  
- **建议**：添加面包屑：设置 > 推理配置

### 47. 侧边栏所有自定义页面图标相同
- **位置**：`Sidebar.tsx` 第 167-176 行  
- **现象**：写作、翻译、知识库、日程、邮件、工具箱全部使用同一个星标图标  
- **建议**：分配不同图标（笔、地球、书本、日历、信封、扳手等）

### 48. 主题切换缺少"跟随系统"选项
- **位置**：`theme-toggle.tsx` 第 11 行  
- **现象**：只在 dark/light 之间循环，无法选择 system  
- **建议**：循环 light → dark → system → light，各状态显示不同图标

### 49. "System"主题不响应 OS 主题变化
- **位置**：`theme.ts`  
- **现象**：`resolve()` 只在启动时读取一次 `prefers-color-scheme`，无 `matchMedia` 监听  
- **建议**：添加 `addEventListener('change', ...)` 实时响应

### 50. Memory Panel 搜索显示计数但不显示结果
- **位置**：`MemoryPanel.tsx` 第 79-83 行  
- **现象**：显示"匹配 X / Y 条目"但从不渲染 `recallResults` 数组  
- **建议**：在搜索框下方渲染匹配结果列表

### 51. FileTree 目录判断逻辑有缺陷
- **位置**：`FileTree.tsx` 第 52 行  
- **现象**：通过 `!item.includes('.')` 判断是否为目录，`.github`、`.vscode` 等带点目录被误判为文件  
- **建议**：后端返回 isDir/isFile 元数据

### 52. FileTree 只渲染两层
- **位置**：`FileTree.tsx` 第 50-89 行  
- **现象**：第三层及更深的目录不可展开  
- **建议**：实现递归 TreeNode 组件

### 53. Git 面板不显示变更文件列表
- **位置**：`GitPanel.tsx`  
- **现象**：只显示分支和 dirty/clean 状态，不显示具体哪些文件改了  
- **建议**：添加暂存/未暂存文件列表 + 快捷操作

### 54. BottomBar 无连接状态指示

## 修复优先级建议

| 优先级 | 数量 | 预计工时 | 核心价值 |
|--------|------|----------|----------|
| **P0** | 8 项 | 3-5 天 | 消除致命的数据丢失、功能失效、焦点劫持 |
| **P1** | 17 项 | 7-10 天 | 补全核心交互（消息操作、搜索、会话管理、快捷键） |
| **P2** | 38 项 | 10-15 天 | 消除各页面痛点、提升工作流连贯性 |
| **P3** | 55 项 | 15-20 天 | 全面打磨、可访问性、i18n、细节优化 |

> **建议先集中解决 P0 + P1（约 25 项），预计 2 周可完成，用户体验将有质的飞跃。**
