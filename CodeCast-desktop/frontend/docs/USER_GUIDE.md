## 🚀 快速入门

### 第一步：安装与启动

#### Windows
1. 从 [GitHub Releases](https://github.com/1405451216/CodeCast/releases) 下载 `CodeCast.exe`
2. 双击运行安装程序
3. 安装完成后，从桌面或开始菜单启动 CodeCast

#### macOS (Apple Silicon / Intel)
1. 下载对应的 `.dmg` 文件
2. 拖拽到 Applications 文件夹
3. 从 Launchpad 启动

### 第二步：配置 AI 提供商

首次启动后，你需要配置至少一个 AI 提供商：

1. 点击左下角 **⚙️ 设置** 图标（或按 `Ctrl + ,`）
2. 进入 **模型管理** 标签页
3. 选择提供商并填入 API Key：

| 提供商 | 获取 API Key | 推荐场景 |
|--------|-------------|---------|
| **DeepSeek** | [platform.deepseek.com](https://platform.deepseek.com) | 🌟 推荐！高性价比，中文优化 |
| **OpenAI** | [platform.openai.com](https://platform.openai.com) | GPT-4o 全能模型 |
| **Anthropic** | [console.anthropic.com](https://console.anthropic.com) | 最强推理能力 |
| **Google** | [aistudio.google.com](https://aistudio.google.com) | 百万上下文窗口 |
| **Ollama** | 无需 API Key | 本地运行，完全离线 |

4. 选择默认模型
5. 点击 **保存**

### 第三步：开始你的第一次对话

🎉 **恭喜！你现在可以开始使用了！**

在输入框中输入你的第一个问题：

```
帮我创建一个 React 计数器组件
```

按 **Enter** 或点击发送按钮，AI 将立即为你生成代码。

## 🔥 核心功能

### 1. AI 对话

CodeCast 的核心是智能对话系统，支持多种交互方式：

#### 基础对话
直接输入自然语言描述你的需求：

```
# 示例 1：生成代码
用 TypeScript 写一个防抖函数

# 示例 2：解释代码
这段代码的作用是什么？[粘贴代码]

# 示例 3：调试错误
我的 React 组件报错 "Cannot read property 'map' of undefined"

# 示例 4：重构建议
这个函数太长了，帮我拆分成更小的函数
```

#### 高级特性

**🎯 斜杠命令**
输入 `/` 触发快速命令菜单：
- `/code` - 代码生成模式
- `/review` - 代码审查模式
- `/debug` - 调试助手模式
- `/doc` - 文档生成模式

**🧠 深度思考模式**
开启后，AI 会进行更深入的推理：
- 设置 → 对话设置 → 开启"深度思考"
- 适合复杂算法设计、架构决策

**📏 长上下文模式**
支持 1M token 的超长上下文窗口：
- 可一次分析整个大型项目
- 适合代码库迁移、全局重构

**🎭 人格风格切换**
- 友好型：轻松活泼，适合学习
- 专业型：严谨详细，适合开发
- 简洁型：直奔主题，效率优先
- 详细型：全面深入，彻底理解

### 2. 情景记忆系统

这是 CodeCast 最独特的能力——**跨会话的连续性体验**。

#### 工作原理

```
你说话 → AI 生成摘要和标签 → 存入 SQLite 数据库 
                                    ↓
新对话开始 → 自动检索相关记忆 → 注入上下文 → AI "记得"你之前说过什么
```

#### 实际效果

**没有情景记忆：**
```
用户: 帮我用 React 写一个登录页面
AI:  [生成通用登录页面]

用户: （新会话）修改一下，要符合我们项目的暗色主题
AI:  ❓ 什么项目？什么暗色主题？
```

**有情景记忆：**
```
用户: 帮我用 React 写一个登录页面
AI:  [生成登录页面] ✅ 记住：React 项目，暗色主题

用户: （新会话）修改一下，要符合我们项目的暗色主题
AI:  ✅ 根据之前的对话，你的项目使用深色背景 (#1a1a2e)，主色调为紫色...
    [生成符合主题的版本]
```

#### 使用技巧

1. **主动提供上下文**
   ```
   我的项目是一个电商后台管理系统，使用 React + TypeScript + Tailwind CSS
   ```

2. **使用标签系统**
   - AI 会自动为对话打标签
   - 你也可以手动添加标签方便检索

3. **查看记忆统计**
   - 点击侧边栏的记忆图标
   - 查看活跃度、热门标签、时间线

### 4. Agent 自主执行

Agent v2.0 是 CodeCast 的杀手锏功能——**让 AI 真正帮你写代码**。

#### 如何触发 Agent 模式

**方法 1：自动检测**
当你提出需要执行的任务时，AI 会自动进入 Agent 模式：
```
帮我重构 Login 组件，将表单验证逻辑提取出来
```

**方法 2：手动开启**
点击输入框上方的 **🤖 Agent** 按钮

#### Agent 执行流程

```
用户输入任务
    ↓
AI 分析并拆解子任务
    ↓
┌─────────────────────────────┐
│ Step 1: 读取 Login.tsx      │ ← 自动选择工具
│ Step 2: 创建 validator.ts   │
│ Step 3: 更新 Login.tsx      │
│ Step 4: 运行测试            │
└─────────────────────────────┘
    ↓
每个步骤执行前展示计划 ✓
    ↓
高危操作（写文件/删除）暂停等待确认
    ↓
显示 Diff 变更预览
    ↓
用户批准/拒绝
    ↓
继续下一步或修复错误
```

#### 内置工具集（17 个）

| 类别 | 工具 | 说明 |
|------|------|------|
| **文件操作** | ReadFile / WriteFile | 读取/写入文件 |
| **终端** | RunCommand | 执行 Shell 命令 |
| **Git** | GitCommit / GitStatus | Git 操作 |
| **测试** | RunTest | 执行单元测试 |
| **搜索** | SearchCode | 代码搜索 |
| **网络** | WebSearch / FetchURL | 网络请求 |
| **交互** | AskUser | 向用户确认 |

#### Checkpoint 安全机制

对于高危操作，Agent 会自动暂停：

```
⚠️ 即将执行高危操作：删除 src/utils/deprecated.ts

变更预览：
━━━ src/utils/deprecated.ts ━━━
- line 1-50: 整个文件将被删除

风险等级：🟡 中等
原因：该文件被 3 个其他文件引用

[✅ 批准] [❌ 拒绝] [👁️ 查看引用]
```

### 5. 代码补全

CodeCast 提供三级智能代码补全：

#### L1: 本地缓存（< 50ms）
- 最近使用的代码片段
- 基于当前文件的上下文

#### L2: AI 云端补全（< 500ms）
- 使用 DeepSeek/OpenAI 模型预测
- 支持 FIM（Fill-in-the-Middle）模式

#### L3: 项目符号索引（< 10ms）
- 自动索引函数名、变量名、导入路径
- 跨文件引用解析

#### 使用方法

```typescript
// 输入以下代码时：
import { useState, useEffect } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  
  // 光标在此处按 Tab → AI 补全：
  // useEffect(() => {
  //   document.title = `Count: ${count}`;
  // }, [count]);
  
  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}
```

**快捷键**:
- `Tab`: 触发补全
- `Ctrl + Space`: 强制刷新
- `↑↓`: 选择候选项
- `Esc`: 关闭补全

### 6. Git 工作流

#### AI Commit Message

修改完代码后，让 AI 为你生成规范的 commit message：

1. 在 Git 面板中查看已修改的文件
2. 点击 **🤖 AI Commit** 按钮
3. AI 分析 diff 后生成 message（遵循 Conventional Commits 规范）
4. 一键应用或手动调整

**示例输出**:
```
feat(auth): add JWT token refresh mechanism

- Implement automatic token refresh before expiration
- Add error handling for failed refresh attempts
- Update auth interceptor to use new tokens

Closes #123
```

#### PR 助手

创建 Pull Request 时：
- 自动填充标题和描述
- 关联相关 Issue
- 生成 Change Log

#### 冲突解决

遇到合并冲突时：
1. AI 检测冲突文件
2. 提供三种解决方案：
   - 🅰️ 采用我们的更改
   - 🅱️ 采用他们的更改
   - 🔀 AI 智能合并
3. 预览合并结果后再确认

### ⏰ 自动化任务

创建定时任务实现无人值守运行：

```yaml
# 示例 1：每日代码审查
name: 每日代码审查
schedule: daily 09:00
action: |
  检查 src/ 目录下所有 .ts 和 .tsx 文件，
  识别潜在的性能问题和安全漏洞，
  生成审查报告发送通知。

# 示例 2：每周备份
name: 每周数据备份
schedule: weekly Monday 08:00
action: 导出所有对话记录和记忆数据到 backup/
```

**支持的调度表达式**:
- `every 30m` - 每 30 分钟
- `hourly` - 每小时
- `daily` - 每天（默认 00:00）
- `daily 09:00` - 每天 9 点
- `weekly Monday` - 每周一
- `0 */2 * * *` - Cron 表达式（每 2 小时）

## ❓ 常见问题

### Q1: API Key 安全吗？

✅ **绝对安全！**
- API Key 使用 AES-256-GCM 加密存储在本地
- 永远不会上传到我们的服务器
- 只有你主动发送给 AI 的内容才会通过 API 传输

### Q2: 可以离线使用吗？

可以！如果你使用 **Ollama** 本地模型：
1. 安装 Ollama：https://ollama.ai
2. 下载模型：`ollama pull codellama:13b`
3. 在 CodeCast 中选择 Ollama 提供商
4. 完全离线使用，零成本！

### Q3: 支持哪些编程语言？

支持所有主流编程语言：
- **前端**: JavaScript / TypeScript / React / Vue / Angular
- **后端**: Python / Go / Java / Node.js / Rust
- **移动端**: Swift / Kotlin / Flutter
- **其他**: SQL / Shell / Docker / YAML ...

### Q4: 数据存储在哪里？

所有数据都存储在本地：
- **Windows**: `%APPDATA%/CodeCast/`
- **macOS**: `~/Library/Application Support/CodeCast/`
- **Linux**: `~/.config/CodeCast/`

数据库格式：SQLite（可随时导出）

### Q5: 如何迁移到新电脑？

1. 在旧电脑：设置 → 数据管理 → 导出全部数据
2. 将导出文件传输到新电脑
3. 在新电脑：设置 → 数据管理 → 导入数据

### Q6: 性能如何优化？

如果感觉卡顿，可以尝试：
1. **关闭深度思考模式**（减少计算量）
2. **清理缓存**：设置 → 高级 → 清理缓存
3. **减少记忆保留天数**：默认 30 天可改为 7 天
4. **禁用动画**：设置 → 外观 → 减少动效

```
# ❌ 一次性要求太多
```

```
1. 搜索所有包含 console.log 的文件

4. 生成变更报告
```

### 🔒 安全最佳实践

1. **不要分享敏感信息**
   - ❌ API Keys、密码、Token
   - ❌ 个人身份信息（PII）
   - ❌ 生产环境配置
